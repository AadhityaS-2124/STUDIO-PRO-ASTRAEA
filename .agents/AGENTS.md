# Electron Media Streaming & File Handling Rules

- **File URL Construction**: Never manually prepend `file://` to local file paths (especially using string templates with `path.join`), as this results in backslashes (`\`) on Windows which Chromium fails to resolve (triggering `ERR_FILE_NOT_FOUND`). Always use Node's `url.pathToFileURL(filePath).toString()` to build clean, cross-platform file URLs.
- **Custom Protocol Media Playback (Range Requests)**: When implementing custom protocols (such as `media://`) for rendering or streaming local videos and audio in Chromium, you must handle HTTP `Range` headers. Media players perform range requests (`Range: bytes=start-end`) to buffer and seek. Return a `206 Partial Content` status response using `fs.createReadStream` with `Content-Range`, `Accept-Ranges: bytes`, and `Content-Length` headers. Returning a simple `200` fetch response will cause black screens or seek locks.
- **MIME-Type Headers for Protocols**: Ensure that custom protocol responses set the correct `Content-Type` header based on the file extension (e.g., `video/mp4`, `audio/wav`, `image/png`). Chromium will fail to decode or play media elements if correct MIME-types are missing.

# Skeuomorphic & Semantic Design Rules

- **Semantic Class Naming Overrides**: When refactoring components that originally utilized inline or utility-first frameworks (e.g., Tailwind), extract clean semantic selector classes (e.g., `.control-bar`, `.time-display`) into the JSX markup. This allows centralizing physical bezel borders, depth shadows, and metallic reflections in component stylesheets without layout conflict or visual regression.
- **Physical Bezel & Tactile Shadows**: Define global CSS custom properties for hardware representation, including multi-stop steel gradients (`--bg-brushed-metal`), glowing digital cover covers (`--bg-lcd-screen`), and tactile button shadows (`--shadow-button-tactile`).
- **Mechanical Interaction States**: Ensure every mechanical button transitions tactilely: translate vertical offset (`transform: translateY(1px)`) and swap raised borders for a depressed inset (`box-shadow: var(--shadow-button-depressed)`) under the `:active` state to emulate a physical key stroke.
